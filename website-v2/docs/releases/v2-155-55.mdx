# AgentXchain v2.155.55

`v2.155.55` fixes BUG-101: decision objects that use `summary` plus `rationale` now normalize `summary` into the required `statement` field before schema validation. This recovers an otherwise useful tusq.dev full-auto dev turn without manual staging JSON edits.

## Bug Fixes

- **BUG-101: copy decision summary into statement.** Missing `decisions[].statement` now copies from non-empty `summary` after the existing `decision` and `description` sources.
- **Fail-closed missing text.** A decision with no `statement`, `decision`, `description`, or `summary` still fails schema validation.
- **Prompt hardening.** Dispatch field rules now explicitly tell agents not to use `summary` as the decision field name.
