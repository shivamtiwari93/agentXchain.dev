# AgentXchain v2.155.62

`v2.155.62` fixes BUG-108: after BUG-107 recovery, Full Auto Mode now honors terminal blocked results before running post-step paused-active recovery. A real dirty-baseline assignment blocker is surfaced once and left paused instead of being retried in a tight recovery loop.

## Bug Fixes

- **BUG-108: terminal blockers stay terminal.** `executeContinuousRun()` now processes `completed`, `idle_exit`, `failed`, `blocked`, `stopped`, `vision_exhausted`, `vision_expansion_exhausted`, and `session_budget` before invoking the post-step paused-active recovery hook.
- **Preserve BUG-107 startup recovery.** CLI-owned continuous startup still adopts an existing paused or running session with the same vision path, and genuine paused-active runs still recover before work begins.
- **Add command-chain proof.** A new beta-tester scenario runs `agentxchain run --continuous` against a paused active session with a dirty actor-owned file and proves the CLI exits without timeout or post-step recovery looping.
