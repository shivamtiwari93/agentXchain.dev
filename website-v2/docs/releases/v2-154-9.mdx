# AgentXchain v2.154.9

`v2.154.9` is the BUG-52 no-change PM re-verification repair release. It fixes
the remaining tusq.dev quote-back failure where `agentxchain unblock <hesc>`
still redispatched PM after the PM correctly changed no files because the
planning gate artifacts were already complete.

## Bug Fixes

- **BUG-52 no-change gate re-verification**: operator unblock now treats the
  escalation-linked phase entry role as a valid standing-gate source when all
  required human-gated artifacts are already present, the turn passed
  verification, and `proposed_next_role` is `human`. The human unblock is the
  gate approval, even when `files_changed: []`.

- **BUG-52 tusq.dev checkpoint proof**: the patched local CLI was verified
  against the tester checkpoint `a6a388e15674efa89d09e4a0e507f14a702540f3`.
  `unblock hesc_0b1b166cd606d86d` advanced `planning -> implementation`,
  marked `planning_signoff` passed, and dispatched `dev`.
