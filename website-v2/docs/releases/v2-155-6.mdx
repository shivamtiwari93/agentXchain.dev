# AgentXchain v2.155.6

`v2.155.6` is a dogfood unblock patch over `v2.155.5`. It fixes the next
BUG-64 validation layer found in tusq.dev: after schema normalization accepted
the PM sidecar, strict `intent_coverage` still ignored the sidecar-backed
`idle_expansion_result` and failed the conditional idle-expansion acceptance
contract.

## Bug Fixes

- **BUG-64 idle-expansion intent coverage**: `accept-turn` now evaluates
  `vision_idle_expansion` conditional acceptance items against the normalized
  `idle_expansion_result`.
- **Conditional branch handling**: when `kind` is `new_intake_intent`, the
  `vision_exhausted` acceptance branch is not applicable, and the selected
  branch is checked for charter, acceptance contract, priority, and VISION
  traceability.
- **Strict contract preserved**: missing sidecars, malformed sidecars, and
  truthy `vision_exhausted` objects under `kind: "new_intake_intent"` still
  fail before acceptance.
