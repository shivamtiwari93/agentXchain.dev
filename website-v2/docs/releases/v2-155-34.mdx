# AgentXchain v2.155.34

`v2.155.34` fixes BUG-85 in the DOGFOOD-100 run: roadmap replenishment
now scopes acceptance to one bounded VISION-backed milestone instead of
requiring a PM turn to semantically cover the full remaining VISION backlog.

## Bug Fixes

- **BUG-85 VISION replenishment acceptance scoping.** When `.planning/ROADMAP.md`
  is exhausted but `VISION.md` has unplanned scope, AgentXchain now generates
  a scoped acceptance item: the new milestone must cite at least one concrete
  `VISION.md` source section. It no longer emits one giant acceptance item
  containing every unplanned heading.
- **Legacy same-session recovery.** Existing paused runs with the old broad
  `Milestone scope derived from VISION.md sections: ...` item now accept when
  the PM cites `VISION.md` and one listed section, preserving useful planning
  work already produced during the dogfood session.
