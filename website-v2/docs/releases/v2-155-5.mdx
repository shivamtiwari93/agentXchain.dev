# AgentXchain v2.155.5

`v2.155.5` is a dogfood unblock patch over `v2.155.4`. It fixes the second
real tusq.dev idle-expansion sidecar shape: PM emitted
`kind: "new_intake_intent"` with a harmless `vision_exhausted: false` sentinel.
The acceptor now drops false/null exhaustion sentinels before strict validation.

## Bug Fixes

- **BUG-64 false-sentinel tolerance**: `accept-turn` accepts top-level
  `idle_expansion_result` objects and sibling `idle-expansion-result.json`
  sidecars that include `vision_exhausted: false` or `vision_exhausted: null`
  with `kind: "new_intake_intent"`.
- **Strict contract preserved**: truthy `vision_exhausted` objects remain valid
  only for `kind: "vision_exhausted"`, and malformed idle-expansion results
  still fail schema validation.
