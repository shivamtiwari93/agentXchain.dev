# AgentXchain v2.155.56

`v2.155.56` fixes BUG-102: passing verification may include deliberate negative-case commands with non-zero exits only when those exits are explicitly declared as expected. This recovers the tusq.dev full-auto QA turn that proved CLI rejection paths with exit-1 checks.

## Bug Fixes

- **BUG-102: expected non-zero verification evidence.** `verification.machine_evidence[].expected_exit_code` can now declare the expected exit code for a negative-case command.
- **Evidence-summary compatibility.** Existing staged results can also pass when `evidence_summary` explicitly names the command or recognized snippet and says it exits with the observed non-zero code.
- **Fail-closed accidental failures.** Non-zero machine evidence without a matching expected declaration still blocks `verification.status: "pass"`.
- **Prompt hardening.** Dispatch instructions still prefer wrapping negative checks in zero-exit verifiers, while documenting the explicit raw non-zero evidence contract.
