# AgentXchain v2.155.57

`v2.155.57` fixes BUG-103: roadmap-replenishment PM turns that emit decision text as `decisions[].title` now normalize that text into the required `statement` field before schema validation. This recovers the tusq.dev full-auto PM turn that derived the next roadmap milestone from remaining VISION scope.

## Bug Fixes

- **BUG-103: decision title statement normalization.** Missing or blank `decisions[].statement` now copies from non-empty `title` after existing `decision`, `description`, and `summary` sources.
- **Fail-closed missing content.** Decisions with no statement source material still fail schema validation; AgentXchain does not synthesize decision text from `rationale` alone.
- **Prompt hardening.** Dispatch instructions now explicitly forbid `title` as the decision statement field name.
