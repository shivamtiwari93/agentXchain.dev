# Governed product examples

These are not stubs, templates, or demos. Each is a **complete, working product** built from scratch under AgentXchain governance with real code, real tests, governed workflow-kit artifacts, and a distinct team shape. They prove that AgentXchain can take a product idea from zero to shippable software across varied domains.

Every example includes:
- Runnable source code with tests
- A governed `agentxchain.json` with explicit roles, phases, and workflow-kit artifacts
- A `TALK.md` capturing the governed collaboration log
- Planning artifacts specific to the product category
- Passing `agentxchain template validate --json` proof

## Consumer SaaS — Habit Board

**A daily habit tracker with streaks, dark-theme UI, and JSON persistence.**

| | |
|---|---|
| **Category** | Consumer SaaS (web app) |
| **Template** | `generic` |
| **Tests** | 29 |
| **Source** | [`examples/habit-board/`](https://github.com/shivamtiwari93/agentXchain.dev/tree/main/examples/habit-board) |

### Team shape

| Role | Why |
|---|---|
| `pm` | Owns user stories, roadmap, and scope decisions |
| `designer` | Owns UX flows, design decisions, and accessibility |
| `fullstack_dev` | Implements backend API, frontend UI, and streak logic |
| `qa` | Writes acceptance matrix and ship verdict |

### Workflow phases

**planning** → **design** → **implementation** → **qa**

The `design` phase is unique to this example — it produces consumer-specific artifacts like user flows, design decisions, and UX patterns before implementation begins.

### Key workflow-kit artifacts

- `user-stories.md` — product requirements from user perspective
- `ux-flows.md` — interaction flows and navigation patterns
- `design-decisions.md` — visual and UX decisions with rationale
- `API_CONTRACT.md` — REST endpoint specification
- `acceptance-matrix.md` — pass/fail test criteria
- `ship-verdict.md` — final go/no-go decision

### Run it

```bash
cd examples/habit-board
npm install
node --test test/          # 29 tests
node src/server.js         # http://localhost:3000
# Validate governance
node ../../cli/bin/agentxchain.js template validate --json
```

## Governed build proof

Each example's governed provenance is proven by:

1. **Git history** — the repo's commit log for each `examples/<name>/` directory shows the governed development process.
2. **`TALK.md`** — each example includes a collaboration log capturing the governed turn-by-turn development.
3. **Workflow-kit artifacts** — the `.planning/` files in each example are real governed artifacts produced during development, not retroactive documentation.
4. **`template validate --json`** — every example passes AgentXchain's governance validation, proving the `agentxchain.json` config, role definitions, and workflow-kit structure are conformant.

This is the provenance contract defined in [`.planning/PRODUCT_EXAMPLES_GOVERNED_PROOF.md`](https://github.com/shivamtiwari93/agentXchain.dev/tree/main/.planning/PRODUCT_EXAMPLES_GOVERNED_PROOF.md).
