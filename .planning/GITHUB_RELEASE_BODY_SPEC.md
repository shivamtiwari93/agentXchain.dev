# GitHub Release Body Spec

## Purpose

Make the GitHub Release object truthfully point at AgentXchain's governed release surface instead of relying on low-signal autogenerated notes that can collapse to unrelated PR noise.

## Interface

- Workflow: `.github/workflows/publish-npm-on-tag.yml`
- Renderer: `cli/scripts/render-github-release-body.mjs`
- Inputs:
  - `--target-version <semver>` required
  - `--repo <owner/name>` optional; defaults to `GITHUB_REPOSITORY` when available
- Output:
  - Markdown release body written to stdout

## Behavior

1. The renderer reads the governed website release page at `website-v2/docs/releases/v<major>-<minor>-<patch>.mdx`.
2. It extracts:
   - the first explanatory paragraph after the `# AgentXchain vX.Y.Z` heading
   - the strongest aggregate evidence line that includes a concrete test count and `0 failures`
3. It emits a compact GitHub Release body containing:
   - public release-notes URL on `agentxchain.dev`
   - npm package URL for the exact version
   - the extracted summary paragraph
   - the extracted evidence line
   - a compare link to the previous semver tag when one exists
4. The publish workflow uses that generated body when creating a new GitHub Release.
5. The publish workflow also updates an existing release body on rerun so stale autogenerated notes are repairable instead of permanent.

## Error Cases

- Missing `--target-version` fails closed.
- Missing release-notes page for the target version fails closed.
- Missing summary paragraph fails closed.
- Missing concrete evidence line fails closed.
- If no previous semver tag exists, omit the compare link instead of failing.

## Acceptance Tests

- `AT-GRB-001`: the renderer fails without `--target-version`.
- `AT-GRB-002`: the renderer output includes the public docs release URL for the target version.
- `AT-GRB-003`: the renderer output includes the npm package URL for the target version.
- `AT-GRB-004`: the renderer output includes the release summary paragraph from the governed release page.
- `AT-GRB-005`: the renderer output includes the aggregate evidence line with concrete test counts.
- `AT-GRB-006`: the publish workflow uses the renderer output when creating a release and updates existing releases on rerun.

## Open Questions

- None. This is intentionally narrow: truthful body generation, not a broader release-announcement templating system.
