# Release Notes Sidebar Spec

## Purpose

Ensure every website release note page under `website-v2/docs/releases/` is presented inside a single collapsible `Release Notes` sidebar category on the docs site instead of leaking out as top-level sidebar items.

## Interface

- Sidebar source: `website-v2/sidebars.ts`
- Release docs directory: `website-v2/docs/releases/`
- Category contract:
  - label: `Release Notes`
  - collapsible: `true`
  - collapsed by default: `true`
  - items: all release note docs generated from `website-v2/docs/releases/`

## Behavior

- The docs sidebar must contain exactly one top-level `Release Notes` category entry.
- Every release note page in `website-v2/docs/releases/` must be nested under that category.
- The release notes category must be collapsed by default on the live docs site.
- Adding a new release note file under `website-v2/docs/releases/` must automatically place it inside the `Release Notes` category without requiring a hand-maintained sidebar entry.

## Error Cases

- A bare top-level `autogenerated` sidebar entry for `releases` is not acceptable because it can flatten release docs into top-level items.
- Duplicate `Release Notes` categories are not acceptable.
- A category that is expanded by default is not acceptable.
- A release note file existing on disk but missing from the generated release category is not acceptable.

## Acceptance Tests

- `AT-RNS-002`: the `Release Notes` category is `collapsible: true`.
