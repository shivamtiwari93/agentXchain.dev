Docs / Plugins

Package hook integrations without forking core config.

Plugins are governed hook bundles with a manifest, an install lifecycle, and clean removal. They run under the same trust model and tamper protections as hand-written hooks. No marketplace, no sandboxing, no magic.

Plugin manifest

Every plugin root must contain an agentxchain-plugin.json file:

{
  "schema_version": "0.1",
  "name": "@agentxchain/plugin-slack-notify",
  "version": "1.0.0",
  "description": "Posts governed lifecycle notifications to Slack.",
  "hooks": {
    "after_acceptance": [
      {
        "name": "slack_notify_acceptance",
        "type": "process",
        "command": ["node", "./hooks/after-acceptance.js"],
        "timeout_ms": 3000,
        "mode": "advisory"
      }
    ]
  },
  "config_schema": {
    "type": "object"
  }
}
Field Required Description
schema_version Yes Must be "0.1" for phase 1 plugins
name Yes Package-style identifier (e.g. @org/plugin-name or my-plugin)
version Yes Semantic version string
hooks Yes Object mapping lifecycle phases to hook arrays. Same shape as agentxchain.json hook config.
description No Human-readable description
config_schema No JSON Schema for plugin config. Recorded but not enforced in phase 1.

Installing plugins

agentxchain plugin install <path|npm-package>

Install resolves the source, validates the manifest, checks for hook-name collisions, copies the plugin into .agentxchain/plugins/<derived-id>/, rewrites relative command paths, and merges hooks into agentxchain.json.

Three source types

Source Example How it works
Local directory agentxchain plugin install ./my-plugin Reads agentxchain-plugin.json from the directory root
Local archive agentxchain plugin install ./my-plugin.tgz Extracts .tgz/.tar.gz and reads the manifest
npm package spec agentxchain plugin install @agentxchain/plugin-slack-notify Runs npm pack to download, then extracts and reads the manifest

Path rewriting

Command tokens beginning with ./ or ../ are rewritten to project-relative paths under .agentxchain/plugins/<id>/ so hook execution remains valid after copy. Absolute paths and bare commands are not rewritten.

Collision protection: If any plugin hook name already exists in the same lifecycle phase of agentxchain.json, the install is rejected with no config or filesystem mutation. This is fail-closed by design.

Listing plugins

agentxchain plugin list [--json]

Reports each installed plugin's name, version, install path, source type, declared hook bindings by phase, and whether the install directory still exists on disk.

Removing plugins

agentxchain plugin remove <name>

Removal is metadata-driven. It deletes only the hook names recorded for that plugin, preserves unrelated operator-defined hooks, removes the plugin metadata entry from agentxchain.json, and deletes the installed plugin directory under .agentxchain/plugins/.

  • Removal refuses to proceed if the plugin install path points outside .agentxchain/plugins/ (safety check).
  • There is no upgrade command in phase 1. To upgrade, remove then reinstall.

Authoring a plugin

A minimal plugin is a directory with two files:

1. agentxchain-plugin.json

{
  "schema_version": "0.1",
  "name": "my-notify-plugin",
  "version": "0.1.0",
  "hooks": {
    "after_acceptance": [
      {
        "name": "my_notify",
        "type": "process",
        "command": ["node", "./notify.js"],
        "timeout_ms": 5000,
        "mode": "advisory"
      }
    ]
  }
}

2. notify.js

// Hook receives the turn payload on stdin as JSON
const chunks = [];
process.stdin.on('data', c => chunks.push(c));
process.stdin.on('end', () => {
  const payload = JSON.parse(Buffer.concat(chunks).toString());
  console.log(`Turn accepted: ${payload.summary}`);
  // Exit 0 = allow, exit 1 = warn (advisory) or block (blocking)
});

Test it locally

# From a governed project directory
agentxchain plugin install ./path/to/my-notify-plugin
agentxchain plugin list
agentxchain step --role pm
# Hook fires after acceptance

Distribute as npm package

# Publish to npm
cd my-notify-plugin
npm init -y  # if no package.json
npm publish

# Install from npm
agentxchain plugin install my-notify-plugin

Failure modes

Scenario Behavior
Non-governed project All plugin commands rejected
Missing agentxchain-plugin.json Install rejected
Invalid manifest shape Install rejected with field-level errors
Hook name collision Install rejected, no config or filesystem mutation
Plugin already installed Install rejected
npm pack failure Install rejected with captured stderr context
Partial install failure Staged plugin directory cleaned up, config unchanged
Unsafe removal path Removal rejected (path must be under .agentxchain/plugins/)
