Philosophy

Mandatory challenge, explicit gates, auditable delivery

Most multi-agent systems delegate from one brain to sub-agents. AgentXchain inverts this: each agent has its own mandate and is required to challenge the others. The orchestrator enforces turn-taking, validates results, and gates phase transitions — it doesn’t make the decisions.

Mandatory challenge

Every turn must identify at least one risk or objection about the previous agent’s work. Blind agreement is a protocol violation. This is adversarial robustness applied to software delivery.

Divergent agents, healthy friction

The PM pushes for user value. The dev pushes for feasibility. QA pushes for correctness. The eng director pushes for quality. They’re supposed to disagree. That tension produces better software than any single agent ever could.

Real-world pressure to ship

Phases, turn limits, mandatory verification, human checkpoints. The team can’t bikeshed forever. Constraints force convergence on a real product — just like a deadline does for human teams.

How real teams work, applied to AI

In a real software company, the PM doesn’t write code. The dev doesn’t decide what to build. QA doesn’t trust anyone else’s test results. And there’s always a moment where someone says “this isn’t good enough” and sends it back.

That’s what AgentXchain recreates. Not the org chart. Not the meetings. The structured disagreement that forces one agent’s blind spot into another agent’s review surface.

Prompts and protocol text require agents to challenge prior work. The CLI enforces turn routing, handoffs, and artifact checks; model behavior still depends on following those rules.

How it works

Structured turns. Mandatory challenge. Human authority.

The workflow is optimized for convergence quality, not generation speed.

1

Init

Run agentxchain init --governed. Define roles, phases, adapters, and gates in agentxchain.json.

2

Step

agentxchain step assigns a turn. The agent works within its mandate and produces a structured turn result.

3

Gate

Orchestrator validates artifacts, enforces challenge requirements, and checks phase gates. Human approves transitions.

4

Ship

QA requests completion. Human makes the final ship/no-ship decision. Every turn is in the audit trail.

Quality

Protocol checks that keep review honest

AgentXchain does not pretend a review model equals a test runner. It enforces review boundaries, challenge requirements, and evidence-backed acceptance before the run can advance.

Mandatory objections

Review turns must raise at least one objection. Blind approval is a protocol violation, so QA cannot rubber-stamp the previous turn and call it done.

Evidence, not vibes

Verification claims are recorded as structured machine evidence. The orchestrator checks files, gate artifacts, and turn-result compliance before acceptance.

Review-only boundaries

QA and leadership roles can challenge, document, and escalate, but they cannot silently edit product code. Write authority is part of the contract.

Project structure

The governed scaffold is the working surface

A governed project starts with real protocol artifacts: orchestrator state, append-only history, turn-scoped dispatch bundles, staged results, and planning documents the phase gates actually inspect.

.agentxchain/state.json

The orchestrator-owned source of truth for phase, active turns, gate status, and run completion.

dispatch/ + staging/

Each turn gets a dispatch bundle and a matching staged turn-result.json. This is the handoff contract.

.planning/ gate docs

PM_SIGNOFF.md, ROADMAP.md, acceptance-matrix.md, and ship-verdict.md are checked by the phase gates.

Product outcomes

Why teams adopt AgentXchain

The value is not "more agent activity." The value is better product decisions and fewer expensive misses.

Errors caught earlier
Mandatory challenge surfaces risks before they compound across agent turns.
Higher quality releases
QA evidence and acceptance matrices make release readiness explicit.
Real accountability
Every turn is logged, validated, and attributable to an agent role.
Human control
The protocol requires human approval for phase transitions and run completion, and humans unblock states agents cannot resolve.
Structured convergence
Phases, turn limits, and gate approvals force the team toward a shippable result — no bikeshedding.
Complete audit trail
Every turn records decisions, objections, risks, and verification evidence in machine-readable format.
Portable workflow
Manual, local CLI, and API-backed runtimes can all participate under the same protocol contract.
Faster decision loops
Healthy friction surfaces risks early instead of after launch.
