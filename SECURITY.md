# Security Policy

## Supported Scope

Security reports are welcome for:

- The CLI under `cli/`
- Repository-level scripts and generated hook files
- The example apps under `examples/`

Please include the affected path, impact, reproduction steps, and any required environment details.

## Reporting

Do not open public issues for active vulnerabilities.

Report privately through GitHub Security Advisories: **[open a draft security advisory](https://github.com/shivamtiwari93/agentXchain.dev/security/advisories/new)**. This is the canonical private channel — the maintainers are notified directly and the report stays confidential until a fix ships.

When reporting, include:

- A concise summary
- Severity and likely impact
- Exact files or commands involved
- Minimal reproduction steps
- Suggested mitigation if you have one

## Disclosure

- Reports will be triaged privately first.
- Fixes should land before full public disclosure when feasible.
- If a report affects example code only, disclosure should still clearly say that the example is not production-safe until patched.
